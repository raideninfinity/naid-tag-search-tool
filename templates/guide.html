How to Use

  1. Searching

    • To search, hit Enter key or press the button.
    • Click on to show the settings. Click again to close the menu.
    • Settings:
      • Limit: Number of tags in one page. (20 to 1000)
      • Min Power: Hide tags with power lower than this value. (0 to 10000).
      • Max Power: Hide tags with power higher than this value (1 to 10000).
      • Filter: Hide tags that are not in Danbooru or not in NAID Tag Suggest.
    • Click on Apply button to close the menu and perform a search.
    • Please note that this tool only supports NovelAI Anime Diffusion version 3.

  2. Tag Listing

    • Tags are listed in a table by page. Click on or to change page.
    • Click on Current Page / Total Pages to jump to a page.
    • These fields are shown in the table:
      • Tag Name
        • The first column lists the tag's name.
        • Tags are colored based on Danbooru category: general, artist, copyright, character, meta, and none.
        • Click on to copy the tag to clipboard.
        • For Danbooru tags, the tag name links to the wiki page. Note: The wiki page may or may not exist.
      • Tag Category
        • The second column lists the tag's Danbooru category.
        • If the tag is present in u/Carlyone's NovelAI Tag Experiments, it is marked with . Mouse over the icon to see the details.
      • Tag Power
        • The third column lists the tag's power.
        • means that NovelAI's Tag Suggest system has the tag, and its count is shown. This value represents the knowledge the model has about the tag, and shows as the white circle in the official UI's tag suggest feature. It is reliable.
        • means that NovelAI's Tag Suggest system does not have the tag, so the number of images on Danbooru is shown instead (capped at 10000). This might be due to the training data not having the tag, being aliased to something else, or blocked from suggestion (NSFW and artist tags especially). It is not reliable so take this value with a fine grain of salt.
        • Mouse over the icon to see both values. The full value of will be shown.
        • The training data cut off date is around September 2023 for NAIDv3. The Danbooru image count was scraped around 17 to 20 November 2023.
    • Tags are ordered by count followed by count.

  3. Prefixes

    • You can add prefixes to filter by category.
    • The format is prefix: term. The space after : is optional.
    • The list of supported prefixes are:
      • Danbooru tag categories: general, artist, copyright, character, meta, and none
      • Short form: g, a, r, c, m, and 0.
      • NAIDv3 custom tags: quality, aesthetic, and year.
      • Short form: q, e, and y.
      • Tag Groups: Read the Tag Groups section for more info.
    • Example of prefix usage: character: blue archive, g:red, q:.

  4. Suffixes

    • You can add suffixes to the search term to change the word matching method.
    • The # suffix forces exact tag match. Example: short# can only match short.
    • The @ suffix checks if the tag starts with the search term. Example: color@ matches colored skin but not spot color or multicolored eyes. Adding ^ before the search term has the same effect. Example: ^eye is the same as eye@.
    • The $ suffix checks if the tag ends with the search term. Example: color$ matches spot color but not colored skin or multicolored eyes.
    • The % suffix activates word matching. Example: short% matches short hair but not shorts.
    • Suffixes cannot be combined. Only one will be used, checked in order as listed above.
    • Prefixes can be used together. Example: copyright:^blue, g: man%, and c:eye$.

  5. Tag Groups

    • When z/ or g/ is searched, Tag Group Mode is activated.
    • Try it yourself. You'll learn how it works as you use it more.
    • z/ is related to NovelAI Tag Experiments.
    • g/ is basically Danbooru Tag Groups.
    • The button explores subgroups of g/ tag groups.
    • The or buttons show the tags listed under this tag group. It basically adds : and makes it a prefix. Example: z/verbs:, g/body/face/emotions:, and g/attire:
    • The button is used to go back one group level or go back from tag listing to tag group exploring.
